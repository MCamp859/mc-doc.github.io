Set up Publish directory and GitHub pages repository¶

Once the template repository is set up on GitHub, it is time to build and publish.

Note

We will be using the same working directory for publishing your documentation as we did in the setup guide: otc-tcs-test.

Create the GitHub Pages Repository¶

  1. Create a new repository on GitHub.com by clicking the new button from the project repositories tab.

  2. Name your GitHub pages repository according to the following convention <yourproject>.github.io and click create repository.

  3. On the landing page for the empty repository click the hyperlink labeled README.

  4. Change the name of the file to .nojekyll. There is no need to change the contents of the file.

  5. Scroll down and edit the commit message and click Commit new file.

Clone your GitHub pages repository to your local working directory¶

  1. Create a directory called website inside of otc-tcs-test.

    user@yourcomputer:~/otc-tcs-test$ mkdir website

    At this point, the contents of your otc-tcs-test directory should look like this:

    user@yourcomputer:~/otc-tcs-test$ ls -al
total 0
drwxrwxrwx 0 root root 512 Jul 29 16:54 .
drwxrwxrwx 0 root root 512 Jul 27 16:46 ..
drwxrwxrwx 0 root root 512 Jul 27 16:49 doc-repo
drwxrwxrwx 0 root root 512 Jul 29 16:54 website

  2. Click Clone or download on the repository <> Code tab.

  3. Copy the URL of the respository by clicking the clipboard icon.

  4. From the otc-tcs-test directory clone the GitHub pages repository to your website directory.

    user@yourcomputer:~/otc-tcs-test$ git clone git@github.com:yourproject/yourproject.github.io.git website

    The contents of otc-tcs-test should now look like this:

    user@yourcomputer:~/otc-tcs-test/$ cd website
user@yourcomputer:~/otc-tcs-test/website$ ls -al
total 0
drwxrwxrwx 0 root root 512 Jul 29 21:43 .
drwxrwxrwx 0 root root 512 Jul 29 16:54 ..
drwxrwxrwx 0 root root 512 Jul 29 21:43 .git
-rwxrwxrwx 1 root root  24 Jul 29 21:43 .nojekyll

For building and deployment instructions go to Installing the documentation tools.
